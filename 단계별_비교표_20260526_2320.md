# 🦉 노후 시뮬레이터 단계별 입력 및 노출 항목 비교표

본 문서는 노후 시뮬레이터의 현재 버전 기준 단계별(Level 1, 2, 3) 화면 노출 항목 및 입력 필드를 비교 분석한 사양 문서입니다.

* **기록 일시**: 2026년 5월 26일 오후 11시 20분 (KST)

---

## 📊 단계별 비교표

| 입력 및 화면 노출 항목 | 1단계 (초간단 진단) | 2단계 (정밀 진단) | 3단계 (심층 시뮬레이션) |
| :--- | :--- | :--- | :--- |
| **자산 (Asset) 입력** | 총자산 1개 수치만 일괄 입력 | 5가지 분류로 입력<br>(부동산, 현금, 주식, 코인, 기타) | 2단계 분류 + **개별 부동산 카드 등록**<br>(부동산별로 이름, 평가액 각각 관리) |
| **부채 (Debt) 입력** | 총부채 1개 수치만 일괄 입력 | 총부채 1개 수치만 일괄 입력 | **각 부동산 카드의 대출금 및 전세보증금 각각 입력** (부채 총액 자동 합산) |
| **소득 (Income) 입력** | 총소득 1개 수치만 일괄 입력 | **원하는 소득 항목 개별 추가**<br>(예: 국민연금, 개인연금 등 자유 등록) | 2단계 항목 추가 + **부동산 카드에 입력한 월세 수입 자동 합산** |
| **지출 (Expense) 입력** | 총지출 1개 수치만 일괄 입력 | **원하는 지출 항목 개별 추가**<br>(예: 생활비, 관리비 등 자유 등록) | 2단계 항목 추가 + **부동산 카드에 입력한 대출이자 자동 합산** |
| **물가 연동 (📈) 토글** | 없음 (수입 60%, 지출 100% 자동 연동) | **등록한 소득/지출 항목별로 물가 연동 여부 개별 체크 가능** | 2단계와 동일하게 항목별 개별 체크 |
| **미래 이벤트 등록** | 없음 | 없음 | **나이별(년차별)로 이벤트 카드 등록**<br>(부동산 매각, 목돈 지출/수입 등 설정) |
| **시뮬레이션 변수** | 없음 (고정값 적용) | 물가상승률 선택 카드만 제공 | **고급 설정 필드 제공**<br>(현금수익률, 부동산상승률 직접 입력, 의료비 자동증가 체크) |
| **결과 화면 그래프** | **미표시 (요약 수치만 제공)** | 표시 (기본 연도별 자산 선 그래프) | 표시 (**부동산 매각 및 이벤트로 인한 자산의 계단식 변동이 반영된 정밀 그래프**) |
| **입력 요약 모달 (📝)** | 기본 정보 8개 요약 수치만 표시 | 기본 정보 + 자산 분류 + 소득/지출 항목 리스트 표시 | **위 항목 전부 + 개별 부동산 카드 리스트 + 이벤트 카드 리스트 + 고급 설정 값 표시** |
| **총 입력 항목 수 (필드 기준)** | **총 8개**<br>(본인/배우자 나이, 총자산, 총부채, 월수입, 월지출, 안전나이, 안전금액) | **약 30개 내외**<br>(1단계 기본 정보 + 자산분류 6개 + 기본 소득/지출 상세 항목 및 토글 등) | **약 50개 이상** (사용자가 등록하는 부동산 카드 수와 미래 이벤트 카드 수에 따라 무제한 확장) |

## 💡 단계별 핵심 설계 요약

* **1단계 (초간단 진단)**: 자산의 세부 구성이나 소득의 물가 상승 연동 여부를 따지지 않고, 대한민국 시니어 평균 자산 구성비(현금 10%, 부동산 90%)와 소득 변동 비율(국민연금 60% 수준)을 가정해 **30초 만에 대략적인 가늠선**을 잡는 단계입니다.
* **2단계 (정밀 진단)**: 사용자의 자산 포트폴리오(예금 vs 부동산 vs 주식)에 따라 운용 수익률을 다르게 반영하며, 특히 국민연금처럼 매년 오르는 돈과 고정 임대료처럼 물가가 반영 안 되는 돈을 **개별(📈 토글)로 발라내어 계산**하므로 시뮬레이션의 정확도가 비약적으로 올라갑니다.
* **3단계 (심층 시뮬레이션)**: 단순히 현재 상황이 지속되는 가정을 넘어 **"5년 후에 살던 집을 팔고 전세로 옮긴다면?", "10년 후에 자녀 결혼 자금으로 1억 원을 내준다면?"** 같은 실제 인생의 큰 변화 요소를 시뮬레이션에 대입하여 부부가 구체적으로 언제 집을 팔고 언제 은퇴 자금을 융통해야 하는지 **실질적인 계획(Planning)**을 짤 수 있는 최상위 단계입니다.
