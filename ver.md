## v1 — 2026-05-13

**최초 풀버전.** 부엉이 캐릭터 + 타이핑 애니메이션 + 3단계 마법사 통합.

### 기능
- 🦉 SVG 부엉이 캐릭터 (4초마다 깜빡임), 말풍선 + 22ms 타이핑 애니메이션
- **3단계 점진적 입력 (Progressive Disclosure)**
  - **레벨 1 (30초)**: 나이·총자산·월수입·월지출·안전기준
  - **레벨 2 (2~3분)**: 자산을 부동산/현금/주식/가상자산/기타로 세분 + 소득·지출 항목별 + 물가연동 토글
  - **레벨 3 (5분~)**: 부동산 개별 입력 + 미래 이벤트(전세 만기·매각·결혼 등) + 고급 설정
- 결과 화면: 판정(safe/warn/danger), 3개 메트릭, Chart.js 자산 추이 그래프
- 단계별 분기 대화 ("어때요? 만족스러우신가요?" → 더 자세히 OR 충분해요)
- 시니어 친화 디자인: 큰 폰트(18px 기본), 빠른 선택 칩, 명확한 버튼

### 알려진 이슈 (v2에서 수정 예정)
- 결과 화면에 와이프 설득용 헤드라인 카피("여보! 우리 ... 써도 된데~")가 본문 안엔 없음 (페이지 상단 헤더에만 있음)
- 1단계 시뮬레이션의 수입 인플레가 `Math.pow(inflFactor, yearOffset * 0.5)` — 의미 모호
- 부동산 대출 원금 상환 미반영 — 매년 증가하는 평가액 대비 부채는 그대로
- 매각 이벤트(`lump+`)가 발생해도 해당 부동산의 임대료/이자가 계속 들어옴
- 월 순현금흐름 메트릭이 어느 시점(현재 vs 미래) 기준인지 라벨에 없음

## v5 — 2026-05-14

**localStorage 기반 자동 저장/복원 — 디버깅·반복 사용 시 재입력 부담 제거.**

### 변경 사항

#### 1. 저장/복원 인프라
- `STORAGE_KEY = 'retirement_sim_state_v1'`, `STORAGE_VERSION = 1` 상수 (스키마 변경 시 버전 올려서 자동 폐기)
- `DEFAULT_STATE` 상수 분리 — restart/초기화 시 일관적으로 사용
- `saveState()`: localStorage에 `{ version, savedAt, state }` 형태로 저장
- `saveStateDebounced()`: 400ms 디바운스 (연속 타이핑 시 과도한 저장 방지)
- `loadState()`: 페이지 로드 시 복원, 버전 미스매치면 자동 폐기
- `clearStoredState()`: 명시적 클리어
- `isStateMeaningful()`: 의미있는 데이터 여부 체크 (디폴트만 있으면 false)
- try-catch로 감싸서 localStorage 사용 불가 환경(Claude.ai 아티팩트 미리보기 등)에서도 안전

#### 2. UI 추가
- **우상단 "💾 저장됨" 인디케이터**: 저장 시 1.5초 페이드인/아웃
- **이어서 하기 배너** (.resume-banner): 페이지 로드 시 저장 데이터가 있으면 자동 표시
  - 입력 요약 ("63세 · 자산 20억 · 부채 8억 · 월수입 520만 · 월지출 700만")
  - 마지막 저장 시각 + 진행 단계
  - 3개 버튼: **[결과 다시 보기 →]** / [N단계에서 수정] / [처음부터]

#### 3. 저장 트리거 지점
- **명시적 저장** (`saveState()`): 1·2·3단계의 "결과 보기" 버튼 클릭 시
- **디바운스 저장** (`saveStateDebounced()`):
  - 2단계: 소득/지출 항목 변경·삭제·추가 시
  - 3단계: 부동산 항목 변경·삭제·추가, 이벤트 항목 변경·삭제·추가 시

#### 4. 복원 흐름
- `bootstrap()` 함수 신설 — `renderLevel1()` 직접 호출 대신 진입점이 됨
- 저장된 의미 있는 state가 있으면 → `renderResumePrompt()`로 분기
- 사용자 선택:
  - **"결과 다시 보기"** → state 복원 + `renderResult()`
  - **"N단계에서 수정"** → 마지막 진행 단계(properties 있으면 3, incomes 있으면 2, 아니면 1)로 이동
  - **"처음부터"** → localStorage 클리어 + DEFAULT_STATE 복원 + Level 1

#### 5. 기타 정리
- `restartBtn` 핸들러도 `clearStoredState()` 함께 호출 + `DEFAULT_STATE` 복제 사용 (깊은 복사로 incomes/expenses 배열 참조 분리)

### 알려진 한계
- localStorage는 **현재 브라우저의 현재 도메인**에만 저장됨. 다른 브라우저/기기에선 별개. file:// 경로로 열면 path별로 격리됨
- Claude.ai 아티팩트 미리보기는 localStorage 차단 — 다운로드 후 직접 브라우저에서 열어야 작동
- 시크릿 모드/프라이빗 브라우징에선 세션 종료 시 자동 폐기

## v8 — 2026-05-14

**🔥 안전 기준 금액을 "현재가치(real)" 기준으로 재정의 — 시뮬 정확성 핵심 개선.**

### 문제 인식
기존엔 `state.safetyAmount`가 **명목값**으로 비교되었음. 즉 사용자가 "5억"을 입력하면 시스템은 32년 후 명목 5억(구매력으로는 약 2.3억)과 비교 → 와이프가 "5억 있으면 안심"이라 생각한 본래 의도와 어긋남.

### 변경 사항

#### 1. 라벨 명시화
- 기존: "② 그 나이에 얼마 정도 남기고 싶으세요?"
- **신규: "② 그 나이에 얼마 (미래 예측 곤란하므로 지금의 자산가치 기준) 정도 남기고 싶으세요?"** — "(...)" 부분은 골드 컬러로 강조
- 보조 설명 추가: "💡 30년 후 가치는 인플레로 가늠하기 어렵죠. 지금 5억의 구매력으로 환산해서 안심되는 금액을 적어주세요. 시뮬은 이를 자동으로 미래 가치로 환산해 비교합니다."

#### 2. 시뮬레이션 비교 로직
- `safetyAmountNominal = state.safetyAmount × (1 + inflation/100)^(safetyAge - myAge)` 도입
- `safe = assetAtSafety >= safetyAmountNominal` (명목 ↔ 명목 비교)
- `assetAtSafetyReal = assetAtSafety / inflMult` (역환산, 표시용)

#### 3. 결과 화면 표시 보강
- **verdict 헤드라인**: "95세에 자산 **18.5억** *(현재가치 8.4억)* — 안전 기준 통과!"
- **메트릭 카드 (안전기준)**: 메인 = 명목값, 부제 = "현재가치 N억 · 목표 M억"
- **차트 부제**: "안전선은 5억 현재가치를 95세 명목 11.0억으로 환산한 값" 안내
- **차트 안전선**: 마커 위치를 `safetyAmountNominal` 명목값으로 표시 (이전엔 원시 safetyAmount 였음 = 오류)
- **차트 안전선 라벨**: "안전선 (95세 명목 11.0억 = 현재 5억)"

#### 4. 와이프 콜아웃 — 현재가치 기준
- `surplusReal = assetAtSafetyReal - state.safetyAmount` (현재가치 여유분)
- `extraMonthly = surplusReal / yearsToSafety / 12` → **현재 가치 기준 추가 사용 가능 금액**
- 메시지: "여보, 우리 매달 **N만원** 더 써도 괜찮대요 ~"
  - 부제: "지금 생활비에 더해 매달 **오늘 화폐 기준** N만원을 더 누릴 여유"
- 직관성이 핵심: 사용자가 오늘 N만원 더 쓴다는 의미로 받아들이는 게 자연스럽게

#### 5. 입력 내용 보기 모달
- "안전 기준 금액" → "**안전 기준 (현재가치)**"로 라벨 변경
- 보조행 추가: "→ 95세 시점 명목 11.0억" (흐림 처리)

### 검증
- 사용자 시나리오(63세, 안전 5억, 인플레 2.5%, 95세): 환산 곱수 2.204, 미래 명목 11.02억
- 시뮬 결과 명목값이 11억 이상이어야 "통과" — 의미적으로 일치

### 호환성 안내
- 기존 v7까지의 state.safetyAmount는 명목값으로 다뤄졌으나, v8부턴 **현재가치로 재해석됨**

- 이는 의도된 동작 — 와이프의 진짜 안전감각과 일치하도록 보정한 것
